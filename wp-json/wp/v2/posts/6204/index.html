If you use VPSSIM or any other administration script in your VPS or GC VM Instance, you’re probably finding yourself unable to copy or modify anything when connecting to your server using WinSCP.<\/p>\n

The reason is that, by default, your server is configured to only allow key pairing authentication, so in order to login as a regular user, you have to do 2 things. First, enable password authentication<\/em> and secondly, enable Login as Root<\/em><\/p>\n

To connect as Root, you first need to set a password for your Root user (sudo passwd<\/code>), then allow logging in remotely as root. To do so, fire up your \/etc\/ssh\/sshd_config<\/code><\/p>\n

nano \/etc\/ssh\/sshd_config<\/pre>\n
 <\/p>\n
Add a line in the Authentication section of the file that says PermitRootLogin yes<\/code>. This line may already exist and be commented out with a “#”. In this case, remove the “#”.<\/p>\n
# Authentication:\r\n#LoginGraceTime 2m\r\nPermitRootLogin yes\r\n#StrictModes yes\r\n#MaxAuthTries 6\r\n#MaxSessions 10<\/pre>\n
Scroll down a few lines and change this line:<\/p>\n
# To disable tunneled clear text passwords, change to no here!\r\n#PasswordAuthentication yes\r\n#PermitEmptyPasswords no\r\nPasswordAuthentication yes\r\n<\/pre>\n
 <\/p>\n
Save the updated \/etc\/ssh\/sshd_config<\/code> file and restart the SSH server (service sshd restart<\/code>)<\/p>\n
Should be good to go now<\/p>\n","protected":false},"excerpt":{"rendered":"
